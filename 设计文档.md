## 项目背景

**你在做什么：** 浙江丽水地区变电站图像监控运维平台
**数据规模：** 71个变电站，分布在9个县（缙云、丽水、云和、庆元、景宁、松阳、遂昌、青田、龙泉）
**每个变电站台账包含：**
- 站内主要设备（录像机、交换机、光纤收发器、集中电源）
- 摄像头型号及数量（网络球、光纤球、枪机）
- 通道 IP 地址和位置
- 机柜位置、电源开关位置
- 市公司平台IP、省公司平台IP

**最终目标：**
1. 首页统计卡片 + 变电站分布地图
2. 变电站列表（县筛选、搜索）
3. 变电站详情（设备台账 + 摄像头列表）
4. 故障报修 + 故障记录 + 运维归档
5. 故障统计（每月次数、TOP N故障摄像头）

**部署环境：** Windows 10 开发 + 飞牛 OS（fnOS）NAS 部署

## 数据结构说明

**每个变电站有 1 个独立的 Excel 文件**（共71个），分布9个县目录。

Excel 结构（以丽水变.xlsx为例）：
- 第1行：变电站名称
- 第2-5行：站内主要设备
- 第6-8行：摄像头型号数量
- 第9-12行：通道端口信息（市/省平台IP、机柜位置、电源屏柜）
- 第13行以后：通道详情（通道1-N，名称、IP、投产时间）

**密码：** 所有变电站录像机密码统一为 `Txjk@1234`，通过后端接口返回，不写在前端源码里
- 前端点击 [显示] → 调用 `GET /api/stations/<id>/password` → 后端返回密码
- 避免密码暴露在浏览器源码或开发者工具中

**故障报修 IP 查不到处理：**
- 按IP查询时，如果该IP在数据库中不存在，允许填写自由文本的摄像头信息（摄像头名称、型号、位置）
- 提交时 `camera_id` 留空，`camera_ip_free_text` 字段存储自由填写的摄像头信息
- 标记为"未关联摄像头"，后续可手动关联

## 数据库设计

**stations 表（变电站）：**
| 字段 | 类型 | 说明 |
|------|------|------|
| id | INTEGER PRIMARY KEY | 自增ID |
| name | TEXT | 变电站名称，如"220kV丽水变" |
| county | TEXT | 所属县，如"丽水" |
| voltage_level | TEXT | 电压等级，如"220kV" |
| camera_count | INTEGER | 摄像头总数 |
| city_platform_ip | TEXT | 市公司平台IP |
| province_platform_ip | TEXT | 省公司平台IP |
| cabinet_location | TEXT | 机柜位置 |
| power_switch | TEXT | 电源开关位置 |
| latitude | REAL | 纬度（手动补充） |
| longitude | REAL | 经度（手动补充） |
| created_at | TIMESTAMP | 创建时间 |

**equipment 表（站内设备）：**
| 字段 | 类型 | 说明 |
|------|------|------|
| id | INTEGER PRIMARY KEY | 自增ID |
| station_id | INTEGER | 所属变电站ID（外键） |
| type | TEXT | 类型：录像机/交换机/光纤收发器/集中电源 |
| model | TEXT | 型号 |
| manufacturer | TEXT | 生产厂家 |
| quantity | TEXT | 数量/说明 |
| install_date | TEXT | 投产时间 |

**cameras 表（摄像头）：**
| 字段 | 类型 | 说明 |
|------|------|------|
| id | INTEGER PRIMARY KEY | 自增ID |
| station_id | INTEGER | 所属变电站ID（外键） |
| channel | TEXT | 通道号，如"通道1" |
| name | TEXT | 摄像头名称/位置描述 |
| model | TEXT | 型号，如"VC-HX900" |
| ip_address | TEXT | IP地址 |
| install_date | TEXT | 投产时间 |

**fault_reports 表（故障报修）：**
| 字段 | 类型 | 说明 |
|------|------|------|
| id | INTEGER PRIMARY KEY | 自增ID |
| camera_id | INTEGER | 摄像头ID（外键，可为空） |
| camera_ip_free_text | TEXT | 自由填写的摄像头信息（当IP未收录时使用） |
| fault_type | TEXT | 故障类型（无图像/球机失控/画面模糊/录像丢失/其他） |
| description | TEXT | 故障描述 |
| reporter | TEXT | 报修人姓名 |
| report_time | TIMESTAMP | 报修时间 |
| status | TEXT | 状态：'open' / 'closed' |
| handled_by | TEXT | 处理人 |
| handled_time | TIMESTAMP | 处理时间 |
| notes | TEXT | 处理备注（归档内容） |

## API 设计

| 接口 | 方法 | 说明 |
|------|------|------|
| `GET /api/stations` | GET | 返回所有变电站（支持 ?county=筛选） |
| `GET /api/stations/<id>` | GET | 返回单个变电站详情（含摄像头和设备） |
| `GET /api/stations/<id>/password` | GET | 返回该变电站录像机密码（统一返回 Txjk@1234） |
| `GET /api/cameras` | GET | 返回所有摄像头（支持 ?station_id=&county=筛选） |
| `GET /api/cameras/by-ip?<ip>` | GET | 按IP查询摄像头（查不到返回 null） |
| `POST /api/fault` | POST | 提交故障报修 |
| `GET /api/fault/records` | GET | 返回故障记录（支持 ?status=&county=筛选） |
| `PUT /api/fault/<id>` | PUT | 更新故障状态（处理/归档） |
| `GET /api/stats/monthly` | GET | 每月故障次数统计 |
| `GET /api/stats/top-cameras` | GET | TOP N 故障摄像头统计 |

**第1周：** 搭环境 + Python 基础
